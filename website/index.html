The alternative to commercial cloud

The cloud, without the cloud

A community mesh network that can spread across continents with no data centres — it runs on commodity hardware and the systems people already own, and it belongs to the people who run it.

…Make Cloud affordable with community Mesh…

A working internet connection · a generous heart · an unused or lesser-used PC / mobile / tablet — that's all this needs.

$0 metered billing 0 data centres 5% safe default share 📱 phones & laptops 100% yours
Start a mesh Why it exists

a living mesh: laptops, workstations, spare GPUs — anywhere on earth

Why

Commercial clouds concentrate expensive machines in data centres and rent them back to you — metered by the second, priced across hundreds of SKUs, with egress fees that tax you for leaving. CloudLess inverts the model: the machines a trusted group already owns federate into one mesh with replication, failover, and web management built in.

☁️ Commercial cloud

  • Unpredictable metered bills; ~30% spend wasted on idle resources
  • Egress fees — a lock-in tax on your own data
  • Your data lives in someone else's data centre
  • Whole FinOps teams exist just to read the invoice

🕸️ CloudLess mesh

  • Zero marginal cost — the hardware is already yours
  • No egress fees: data moves inside your own mesh
  • Privacy-first: data never leaves group hardware
  • A ledger of who gave and who used — no bills, just balance
Cooperatively owned · privacy-first · cheap hardware first · honest security (defense in depth, never "foolproof") · everything web-managed, setup in seconds.

No masters. As the network grows there are no privileged or central nodes to capture or fail — coordination is by gossip among equals, and information is scattered across the mesh and retrievable from anywhere. No one owns the switch, because there is no switch.

Cloud AI is the first service on the mesh — a standard chat-completions API with routing and failover, where cloud cost and privacy pain are highest. Storage, compute, and data services ride the same mesh next.

Run a node

Three steps. Your computer joins the community mesh, shares only what you allow, and earns you service in return.

Step 1 — Get Cloudless running. Pick your system — we've highlighted the one you seem to be on:

  1. Open Terminal — press ⌘+Space, type Terminal, press Return. A window appears where you can type commands — that's all it is.
  2. Install Go (one-time) — download the macOS installer from go.dev/dl and run it, like any app.
  3. Copy-paste this into Terminal and press Return: 
    git clone https://github.com/asifontheline/cloudless
cd cloudless/cloudless
go build -o cloudless ./cmd/cloudless
./cloudless up
    If your Mac offers to install "command line developer tools" the first time, click Install and re-run the first line.
  1. Open PowerShell — press the ⊞ Windows key, type PowerShell, press Enter. A window appears where you can type commands — that's all it is.
  2. Install the tools (one-time) — paste this and press Enter, then close and reopen PowerShell: 
    winget install Git.Git GoLang.Go
  3. Copy-paste this and press Enter: 
    git clone https://github.com/asifontheline/cloudless
cd cloudless\cloudless
go build -o cloudless.exe ./cmd/cloudless
.\cloudless.exe up
  1. Open a terminal — Ctrl+Alt+T on most desktops, or find Terminal in your applications.
  2. Install the tools (one-time, Go 1.22+): 
    sudo apt install golang-go git     # Debian / Ubuntu
sudo dnf install golang git        # Fedora / RHEL
  3. Copy-paste this and press Enter: 
    git clone https://github.com/asifontheline/cloudless
cd cloudless/cloudless
go build -o cloudless ./cmd/cloudless
./cloudless up

When it starts, it prints a link — open it in your browser. That's your control panel: private to you, on your own machine.

Today this is a one-time copy-paste build (~2 minutes). One-click downloads with signed installers are in-build (#23) — this step will become a single download.

Step 2 — Choose how much to share. It starts at a safe 5% — you won't notice it, your machine stays cool and responsive. Got an idle spare box? Slide it up in the panel, to a maximum of 70% (never 100%). Personal phone? Leave it low.

Step 3 — Use it. That's it — you're on the mesh. Chat with AI right in the panel, or point any standard chat app at the address shown there. If any node drops mid-request, another takes over automatically.

Optional — open your node to the wider internet

You only need this if machines outside your home or office should reach your node directly. Skip it otherwise — everything above works without it.

  • Machine with a public address (e.g. a rented server): nothing to do — it's already reachable.
  • Home internet: in your router's settings, forward port 7946 to your computer — a one-time, ~3-minute setup. (Router pages usually live at 192.168.0.1; to learn your public address, search "what is my IP".)
  • Mobile data or shared-building internet: those networks can't forward ports. The one-click overlay — in-build (#20) — will connect nodes across any network with no router setup at all.

Fine print: the share ceiling is 70%, never 100%, so your machine always stays responsive. On phones a thermal & battery guard pauses sharing if the device warms up or runs low (in-build (#54)). Contribution is metered on the ledger and earns you service in return. Prefer the command line? cloudless up -share-cpu 70% sets your share directly.

Sign in — no passwords to remember

When you open your console the first time, you register in one tap using a passkey — your device's fingerprint, face, or PIN. There's no password to invent, forget, or have stolen.

  1. Register: open http://127.0.0.1:8080/ → Create account → confirm with your fingerprint / face. Done. A secret key is created and stored safely on your device — it never leaves it.
  2. Sign in: next time, just touch the sensor. No typing.
  3. More devices: add your phone or laptop by scanning a code — each gets its own passkey.
  4. Lose a device? Revoke it from any other device; a one-time recovery code (saved at signup) restores access if you lose them all.

Passkeys are the phishing-resistant standard (WebAuthn) built into every modern phone and browser — the same thing banks use. Passwordless sign-in is in-build (#52).

One portal for everything. Once you're signed in, your console is your control panel — the single place to both use the mesh (call AI, manage keys, see what you saved vs a cloud bill) and contribute to it (set your share limits, watch your contribution and earned service on the ledger, see the live network map). Same portal on your laptop and your phone. Nothing else to install, nowhere else to go.

Contribute, licensing & donations

Contribute — in any language

The code, the board, and the roadmap are all public, and you don't need to know Go to contribute. Build in any language against the open API; Python is first-class. Full how-to in the contributor guide.

How code lands: every check-in from anywhere is built, vetted, and tested by CI, reviewed, then merged into main by a queue every 2 hours — parallel work worldwide, and main never breaks.
