Mesh Cloud Offerings Catalog

Replicating the core families of public cloud offerings as a mesh-native, community-operated platform — not vendor branding, but equivalent capability under our own vocabulary, built to run on community hardware rather than only centralized data centers.

Design principles

  • Service families should be usable on community hardware, not only on centralized data centers.
  • Every offering supports locality-aware placement, encryption, failover, and quota-aware use.
  • Names are simple, durable, and distinct from public cloud vendor brands.
  • Each service is composable: compute, storage, network, identity, operations, and AI can be mixed into higher-level products.
